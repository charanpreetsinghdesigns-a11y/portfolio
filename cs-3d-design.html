The Brief

First time touching 3D

This was my first assignment involving 3D design and code. The goal was to build an interactive 3D scene that runs entirely in a web browser — no Unity, no Blender exports to an app, just HTML and a framework called A-Frame. I had no prior experience with 3D environments or WebXR, so this was starting from zero.

The Scene

What I built

A single interactive 3D room with two main objects. Everything — lighting, camera, geometry, textures, animations — is declared in HTML tags inside an <a-scene>.

01 — CUP MODEL
3D Cup (GLTF/GLB)
A pre-made 3D cup model loaded via <a-gltf-model>. On click it slides forward and continuously rotates — both animations triggered by the same click event.
02 — GIFT BOX
Box Built from Primitives
A gift box assembled entirely from A-Frame's <a-box> primitives — base, four walls, lid, and an animated front flap that pivots open on click.
03 — ENVIRONMENT
Textured Room
A floor plane with a table texture, a 360° skybox image wrapping the entire scene, and directional + ambient lighting to give the objects depth and shadow.
04 — INTERACTIVITY
Click-based Animations
A-Frame's built-in animation component handles all motion — startEvents: click triggers position changes, rotations, and flap pivots without a single line of JavaScript.
The Code

Key snippets

The entire scene is ~110 lines of HTML. Here are the two most interesting parts — the cup animation and the gift box flap.

index.html — Cup model with click animations 
<!-- 3D Cup: slides forward + spins on click -->
<a-gltf-model
  src="lib/Cup.glb"
  position="0 0.1 -1.5"
  rotation="0 48 0"
  scale="1.5 1.5 1.5"
  animation__slide="
    property: position;
    to: 0 0.25 0.6;
    dur: 1000;
    easing: easeOutCubic;
    startEvents: click"
  animation__rotate="
    property: rotation;
    to: 0 360 0;
    loop: true;
    dur: 3000;
    easing: linear;
    startEvents: click">
</a-gltf-model>

<!-- Gift box flap — pivots open on click -->
<a-entity
  id="bottomFlapPivot"
  position="0 0.12 1.72"
  animation__open="
    property: rotation;
    to: 90 0 0;
    dur: 800;
    easing: easeInOutQuad;
    startEvents: click">
  <a-box
    position="0 0.6 0"
    width="1.85" height="1.2" depth="0.15"
    src="lib/Gift.jpg">
  </a-box>
</a-entity>

Key Learnings

What I took away

01

3D is just another coordinate system. X, Y, Z positions felt intimidating until I realised it's the same logic as CSS layout — just with a depth axis added. Once that clicked, placing objects in space became intuitive.

02

Primitives can build anything. The gift box has no 3D modelling involved — it's five <a-box> elements carefully positioned. Constraints push creative problem-solving.

03

Declarative 3D lowers the barrier massively. A-Frame lets you write a fully interactive 3D scene in HTML, with zero JavaScript for animations. It changed how I think about what "coding" a visual experience means.
